# 前端接口文档

> UEEA2601 上传模块 — 前端对接用
> 后端地址: `http://localhost:8000`
> CORS 已允许: `http://localhost:5173`

---

## ⚠️ 前置条件

**用户必须先配置自己的 GEE 密钥才能使用自动分析功能。**
没有密钥 → 选择自动模式时会返回错误，提示用户去配置。

---

## 接口总览

| 方法 | 路径 | 说明 |
|------|------|------|
| **GEE 密钥** |||
| POST | `/api/gee-key/save` | 保存 GEE 密钥 |
| POST | `/api/gee-key/verify` | 验证 GEE 密钥 |
| GET | `/api/gee-key/status` | 查询密钥状态 |
| DELETE | `/api/gee-key` | 删除密钥 |
| **系统** |||
| GET | `/api/system/status` | 系统健康检查 |
| GET | `/api/system/quota` | 查询 GEE 配额 |
| **上传 & 计算** |||
| POST | `/api/upload/boundary` | 上传边界 |
| GET | `/api/compute/{task_id}/status` | 轮询计算进度 |
| GET | `/api/compute/{task_id}/report` | 获取分析报告 |
| GET | `/api/compute/{task_id}/code` | 获取 GEE 代码（手动模式） |
| **边界查询** |||
| GET | `/api/boundary/{id}` | 获取边界详情 |
| GET | `/api/boundary/list` | 边界列表 |

## 一、GEE 密钥管理（"GEE配置"页面）

### 1.1 保存密钥

```
POST /api/gee-key/save
Header: Authorization: Bearer <用户token>
Body (JSON):
{
  "service_account": "xxx@project.iam.gserviceaccount.com",
  "key_json": "{ ... 完整的密钥JSON内容 ... }"
}

成功: { "success": true, "message": "密钥已保存，请点击验证" }
失败: { "detail": "密钥 JSON 缺少 private_key 字段" }
```

### 1.2 验证密钥

```
POST /api/gee-key/verify
Header: Authorization: Bearer <用户token>

成功: { "success": true, "status": "valid", "message": "GEE 密钥验证成功" }
失败: { "success": false, "status": "invalid", "message": "密钥无效: ..." }
```

**注意**: 验证需要 3-5 秒，前端加 loading。

### 1.3 查询密钥状态

```
GET /api/gee-key/status
Header: Authorization: Bearer <用户token>

有密钥: {
  "has_key": true,
  "status": "valid",          // unverified | valid | invalid
  "service_account": "xxx@xxx.iam",
  "verified_at": "2026-06-08T15:30:00"
}

无密钥: { "has_key": false, "status": null, "service_account": null }
```

### 1.4 删除密钥

```
DELETE /api/gee-key
Header: Authorization: Bearer <用户token>

{ "success": true, "message": "密钥已删除" }
```

## 三、上传边界

```
POST /api/upload/boundary
Content-Type: multipart/form-data

字段:
  file: <文件>                    GeoJSON / .zip(Shapefile) / .tif(GeoTIFF)
  name: <string>                  边界名称
  compute_mode: <string>          "online" 或 "manual"

Header:
  Authorization: Bearer <用户token>

成功 (200):
{
  "boundary_id": 1,
  "task_id": "a1b2c3d4e5f6",
  "status": "processing",
  "filename": "shanghai.geojson",
  "compute_mode": "online"
}

文件过大 (413):
{ "detail": "文件过大 (120MB)，最大允许 50MB" }

无GEE密钥 (503):
{ "detail": "请先在「GEE配置」页面上传并验证您的 GEE 密钥" }

排队中 (429):
{ "detail": "当前排队任务过多，请在队列位置 2 后重试" }
```

## 七、代码示例

### Mapbox 加载 WMS

```javascript
map.addSource('rsei-layer', {
  type: 'raster',
  tiles: [wmsUrl + '&bbox={bbox-epsg-3857}'],
  tileSize: 256,
});
map.addLayer({ id: 'rsei-layer', type: 'raster', source: 'rsei-layer', paint: { 'raster-opacity': 0.7 } });
```

### 轮询进度

```javascript
async function pollStatus(taskId) {
  const res = await fetch(`/api/compute/${taskId}/status`);
  const data = await res.json();
  updateProgressBar(data.progress);

  if (data.status === 'completed') {
    Object.values(data.wms_urls).forEach(loadWmsLayer);
    const report = await fetch(`/api/compute/${taskId}/report`).then(r => r.json());
    renderReport(report);
  } else {
    setTimeout(() => pollStatus(taskId), 3000);
  }
}
```

### CSV 导出

```javascript
function exportCSV(tableData) {
  const csv = [tableData.columns.join(','), ...tableData.rows.map(r => r.join(','))].join('\n');
  const blob = new Blob([csv], { type: 'text/csv;charset=utf-8;' });
  const link = document.createElement('a');
  link.href = URL.createObjectURL(blob);
  link.download = 'analysis_data.csv';
  link.click();
}
```

## 八、用户使用流程

```
1. 用户进入平台 → 显示"GEE配置"引导页
2. 用户按教程注册 GEE → 创建 Service Account → 下载密钥
3. 用户在平台上传密钥 → 点验证 → ✅ 通过
4. 用户上传边界文件 → 点"一键分析"
5. 后端用用户自己的 GEE 密钥跑计算
6. 前端轮询进度 → 完成后渲染报告页面（地图 + 图表 + 数据表格）
```
